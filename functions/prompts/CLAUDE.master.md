## หมวดที่ 2 — ลักษณะและการดำเนินการ {#หมวดที่-2}

### คำอธิบายรายวิชา

- ต้องสอดคล้องกันทั้งภาษาไทยและภาษาอังกฤษ
- ใช้คำกริยาที่สื่อถึงระดับความคาดหวัง เช่น "วิเคราะห์" "สังเคราะห์" "ประยุกต์ใช้"
- **ห้ามใช้คำว่า "เบื้องต้น" หรือ "พื้นฐาน" สำหรับรายวิชาระดับ ป.โท** เพราะแสดงถึงการตั้งเป้าต่ำเกินไป

### วัตถุประสงค์รายวิชา

แต่ละวัตถุประสงค์ต้องเขียนให้:
1. **วัดได้** — ใช้คำกริยาที่ระบุพฤติกรรมที่สังเกตได้ (ดู [ตาราง Bloom's](#blooms))
2. **เชื่อมโยงกับ CLO** — วัตถุประสงค์แต่ละข้อควรสอดคล้องกับ CLO ในหมวด 3
3. **เหมาะสมกับระดับบัณฑิตศึกษา** — อย่างน้อยระดับ Analyze (L4) ขึ้นไป

### เหตุผลในการพัฒนา/ปรับปรุงรายวิชา (ข้อ 3)

**ต้องระบุเสมอ** ไม่ว่าจะเป็นรายวิชาใหม่หรือรายวิชาที่มีการปรับปรุง

```
ตัวอย่างสำหรับรายวิชาใหม่:
"เปิดสอนครั้งแรก เพื่อเสริมสมรรถนะด้าน [ระบุ] ซึ่งระบุไว้ใน PLO [x]
ของหลักสูตรและยังขาดรายวิชารองรับในปีการศึกษาที่ผ่านมา"

ตัวอย่างสำหรับรายวิชาที่ปรับปรุง:
"ปรับปรุงจากข้อเสนอแนะของคณะกรรมการทวนสอบ ปี 256x
โดยเพิ่มเนื้อหา [ระบุ] และปรับวิธีประเมิน [ระบุ]"
```

### การตรวจสอบความสมดุลระหว่างเนื้อหาและหน่วยกิต

ใช้สูตรตรวจสอบเบื้องต้น:

```
ชั่วโมงปฏิบัติต่อสัปดาห์ x 15 สัปดาห์ = ชั่วโมงรวม
ชั่วโมงรวม / จำนวนหัวข้อหลัก = ชั่วโมงต่อหัวข้อ

หากชั่วโมงต่อหัวข้อ < 3 ชั่วโมง -> พิจารณาตัดเนื้อหาหรือเพิ่มหน่วยกิต
```

- **แนวทางสำหรับ 1(0-2-1):** ไม่ควรมีหัวข้อหลักเกิน 3–4 หัวข้อ
- **แนวทางสำหรับ 2(1-2-3):** รองรับได้ 5–6 หัวข้อหลัก

## หมวดที่ 3 — การพัฒนาผลการเรียนรู้และการประเมินผล {#หมวดที่-3}

> นี่คือหมวดที่สำคัญที่สุดและพบปัญหามากที่สุด ต้องตรวจสอบอย่างละเอียด

### 3.1 การเขียน CLO ที่ดี

CLO ที่ดีต้องประกอบด้วย 3 ส่วน:

```
[คำกริยา Bloom's] + [เนื้อหา/ทักษะ] + [บริบท/เงื่อนไข]

ตัวอย่างที่ดี:
"สังเคราะห์แนวทางแก้ปัญหาคลินิกโดยบูรณาการหลักฐานเชิงประจักษ์
 จากฐานข้อมูล PubMed และ Cochrane"
 ^-- Synthesize (L5) + เนื้อหา + บริบทเครื่องมือ

ตัวอย่างที่ไม่ดี:
"นักศึกษาเข้าใจประเด็นปัญหาทางคลินิก"
 ^-- "เข้าใจ" วัดไม่ได้ + ไม่ระบุบริบท
```

### 3.2 Bloom's Taxonomy — คำกริยาแนะนำสำหรับระดับ ป.โท

| ระดับ | Bloom's Level | คำกริยาแนะนำ | ไม่แนะนำ |
|-------|--------------|-------------|---------|
| L3 Apply | ประยุกต์ใช้ | ประยุกต์, แก้ปัญหา, สาธิต, ปฏิบัติ | เข้าใจ, รู้ |
| L4 Analyze | วิเคราะห์ | วิเคราะห์, แยกแยะ, ตรวจสอบ, เปรียบเทียบ | อธิบาย |
| L5 Evaluate | ประเมิน | ประเมิน, วิพากษ์, ตัดสิน, สังเคราะห์ | — |
| L6 Create | สร้างสรรค์ | ออกแบบ, สร้าง, วางแผน, เสนอแนะ | — |

> **กฎสำหรับ ป.โท:** CLO ต้องอยู่ระดับ L4 ขึ้นไป และอย่างน้อย 1 CLO ต้องถึง L5–L6

### 3.3 การตรวจสอบ PLO–CLO Alignment

**ขั้นตอนที่ 1: ตรวจสอบการครอบคลุม PLO**

ทำตาราง mapping และตรวจสอบว่าทุก PLO ที่หลักสูตรกำหนดให้รายวิชานี้รับผิดชอบ (ตาม มคอ.2) มี CLO รองรับ:

```
PLO ที่ได้รับมอบหมาย -> CLO ที่สร้าง -> น้ำหนักคะแนน
PLO1 (ความรู้)    -> CLO1, CLO2   -> xx%
PLO2 (ลักษณะ)    -> CLO3         -> xx%
PLO3 (ทักษะ)     -> CLO4         -> xx%
PLO4 (จริยธรรม)  -> CLO5         -> xx%
                            รวม = 100%
```

**ขั้นตอนที่ 2: ตรวจสอบ Bloom's Gap**

```
ระดับ PLO (จาก มคอ.2) >= ระดับ CLO >= ระดับ L4
```

หาก PLO อยู่ระดับ Create (L6) แต่ CLO ทุกตัวอยู่ที่ Apply (L3) คือมี Gap ที่ต้องแก้ไข

**ขั้นตอนที่ 3: ตรวจสอบความถูกต้องของ Domain**

| PLO Domain | CLO ที่เหมาะสม | CLO ที่ไม่เหมาะสม |
|-----------|--------------|----------------|
| ความรู้ (Knowledge) | CLO ที่วัดการเข้าถึงและใช้ความรู้ | CLO เกี่ยวกับ teamwork |
| ลักษณะบุคล (Person) | CLO ที่วัดพฤติกรรม, teamwork, leadership | CLO เกี่ยวกับการจำข้อเท็จจริง |
| ทักษะ (Skill) | CLO ที่วัดการใช้เครื่องมือ/เทคโนโลยี | CLO เกี่ยวกับจริยธรรม |
| จริยธรรม (Ethics) | CLO ที่วัดการตัดสินใจเชิงจริยธรรม | CLO เกี่ยวกับความรู้ทั่วไป |

> **ข้อผิดพลาดที่พบบ่อย:** นำ CLO เกี่ยวกับ teamwork/interdisciplinary ไปฝากไว้ใต้ PLO "ความรู้" แทนที่จะอยู่ใต้ PLO "ลักษณะบุคล"

### 3.4 การออกแบบการประเมินผล

**กฎสำคัญ: วิธีประเมินต้องสอดคล้องกับระดับ Bloom's ของ CLO**

| ระดับ CLO | วิธีประเมินที่เหมาะสม | วิธีที่ไม่เหมาะสม |
|----------|-------------------|---------------|
| Apply (L3) | OSCE, DOPS, Mini-CEX | MCQ อย่างเดียว |
| Analyze (L4) | CBD, Case Discussion, Essay | MCQ อย่างเดียว |
| Evaluate (L5) | Literature Review Assessment, Critical Appraisal | MCQ, SAQ |
| Create (L6) | Research Proposal, Portfolio, Case Presentation | MCQ, SAQ |

**กฎสำหรับวิชาปฏิบัติ (Lab-based, 0 ชั่วโมงบรรยาย):**

```
สัดส่วน MCQ <= 20% ของคะแนนทั้งหมด
สัดส่วน Performance-based >= 50%
```

**กฎสำหรับสัดส่วนการประเมินด้านความรู้ (ตามระบบหลักสูตร):**

```
MCQ/SAQ/Essay รวมกัน <= 50% (PLO1, PLO5 เท่านั้น)
```

**แนวทางการกระจายน้ำหนักคะแนนที่สมดุล:**

```
วิชาที่มี 4 CLOs และ 4 PLOs:
- ควรให้แต่ละ PLO มีน้ำหนักอย่างน้อย 10–15%
- PLO ที่เป็นจุดเน้นของวิชา อาจมีน้ำหนักสูงสุด 40–50%
- PLO ที่สำคัญแต่ไม่ใช่จุดเน้น ควรมีน้ำหนักไม่ต่ำกว่า 15%
```

### 3.5 ตัวอย่าง Constructive Alignment ที่ดี

```
PLO1 (Create L6)
  +-- CLO1: สังเคราะห์แนวทางแก้ปัญหา... (Synthesize L5)
        |-- วิธีสอน: CBL + Case Presentation
        |-- วิธีประเมิน: CBD 20% + Case Presentation 15%
        +-- เนื้อหา: Root Cause Analysis, SWOT

PLO2 (Researcher characteristics)
  +-- CLO2: แสดงออกถึงการทำงานร่วมกับทีมสหวิชาชีพ... (Apply L3)
        |-- วิธีสอน: Shadowing + Bedside Teaching
        |-- วิธีประเมิน: 360 Feedback 15% + Faculty Eval 10%
        +-- เนื้อหา: Interprofessional teamwork roles

PLO4 (Research Ethics)
  +-- CLO4: ปฏิบัติตามหลักจริยธรรมการวิจัยและการแพทย์... (Apply L3)
        |-- วิธีสอน: Ethics Debate + Bedside Teaching
        |-- วิธีประเมิน: Ethics Assessment 10% + Reflective Portfolio 10%
        +-- เนื้อหา: Clinical Ethics + Research Ethics (IRB, informed consent)
```

## หมวดที่ 4 — แผนการสอน {#หมวดที่-4}

### โครงสร้างแผนการสอนที่ดี

**1. ตรวจสอบชั่วโมงสอดคล้องกับหน่วยกิต**

```
หน่วยกิต X(A-B-C):
A = บรรยาย -> A x 15 สัปดาห์ = ชั่วโมงบรรยายรวม
B = ปฏิบัติ -> B x 15 สัปดาห์ = ชั่วโมงปฏิบัติรวม
C = ค้นคว้า -> C x 15 สัปดาห์ = ชั่วโมงค้นคว้ารวม

ถ้า A = 0 ห้ามใช้คำว่า "Lecture" ในแผนการสอน
ให้ใช้: "Structured Workshop" / "Practical Session" / "Seminar" แทน
```

**2. ระบุรูปแบบการสอนทุกสัปดาห์ให้ครบ** ไม่ว่าจะเป็น Onsite / Online / Hybrid

**3. ระบุอาจารย์ผู้สอนทุกสัปดาห์** และหากใช้ "คณาจารย์" ให้ระบุรายชื่อในหมายเหตุ

**4. Midterm/Final Exam** ต้องระบุ:
- รูปแบบการสอบ (Written / Practical / Oral)
- CLO ที่ประเมิน
- น้ำหนักคะแนน

**5. Ward/Clinical Rotation** ต้องระบุ:
- สถานที่ rotation
- สัดส่วนอาจารย์ต่อนักศึกษา
- แผน supervision และ debrief

### ข้อผิดพลาดที่พบบ่อย

- ใช้คำว่า "Lecture" ในวิชาที่มี 0 ชั่วโมงบรรยาย
- ไม่ระบุรูปแบบการสอนบางสัปดาห์
- Typo ในชื่อกิจกรรม
- มี Midterm และ Final Exam โดยไม่เชื่อมกับ CLO

## หมวดที่ 5 — เอกสารอ้างอิง {#หมวดที่-5}

### แนวทางการเลือกเอกสารอ้างอิง

**ตำราและเอกสารหลัก (ข้อ 1):**
- ใช้ฉบับล่าสุด (ไม่เกิน 10 ปี หากมี)
- ต้องมีตำราที่ครอบคลุมทุก CLO หลัก
- สำหรับวิชาที่มีทั้ง Clinical Ethics และ Research Ethics ต้องมีตำราครอบคลุมทั้งสองด้าน

**เอกสารและข้อมูลสำคัญ (ข้อ 2):**
- ระบุแหล่งข้อมูลออนไลน์ที่นักศึกษาใช้จริง เช่น UpToDate, NICE Guidelines, WHO Clinical Guidelines
- ระบุฐานข้อมูลที่ใช้ในการค้นคว้า เช่น PubMed, Cochrane, Scopus

### ตัวอย่างตำราแนะนำตามหัวข้อ

| หัวข้อ | ตำราแนะนำ |
|--------|----------|
| Evidence-Based Medicine | Straus et al. (2018). Evidence-based medicine (5th ed.) |
| Clinical Ethics | Jonsen et al. (2015). Clinical ethics (8th ed.) |
| Research Ethics | Emanuel et al. Oxford Textbook of Clinical Research Ethics |
| Interprofessional | Reeves et al. (2010). Interprofessional teamwork |
| Literature Review | Greenhalgh (2019). How to read a paper (5th ed.) |
| Critical Thinking | Gambrill (2011). Critical thinking in clinical practice |

## หมวดที่ 6 — การประเมินและปรับปรุง {#หมวดที่-6}

### กระบวนการที่ต้องดำเนินการ (บังคับ)

```
[v] แบบประเมินอาจารย์ผู้สอน
[v] แบบประเมินรายวิชา
[v] มีการตั้งคณะกรรมการสำนักวิชาดำเนินการทวนสอบ
[v] มีการแต่งตั้งผู้ทรงคุณวุฒิร่วมทวนสอบ
[v] สรุปผลทวนสอบผ่านการรับรองจากคณะกรรมการสำนักวิชา  <- สำคัญที่สุด
```

> **ข้อกำหนดบังคับ:** รายการสุดท้าย (การสรุปผลผ่านคณะกรรมการ) ต้องดำเนินการทุกภาคการศึกษา ถือเป็นข้อกำหนดของระบบประกันคุณภาพ (สป.อว./AUN-QA) หากไม่ดำเนินการจะส่งผลต่อการประเมินคุณภาพหลักสูตร

### การเพิ่มกลไก Feedback สำหรับวิชาคลินิก

สำหรับวิชาที่มี Ward Rotation หรือ Clinical Component ควรเพิ่ม:
- Exit Interview / Focus Group Discussion กับนักศึกษาหลัง Rotation
- แบบประเมินจากอาจารย์พี่เลี้ยงทางคลินิก (Clinical Supervisor Feedback)
- Student Reflective Log เพื่อติดตามพัฒนาการ

## ส่วนที่ 1 — การประเมินผลและการตัดเกรด (Grading Evaluation & Suggestion)

**เอกสารที่ใช้:** มคอ.3 + ใบคะแนน/Grade Entry

### 1.1 สรุปข้อมูลเบื้องต้นจากใบคะแนน

ให้สร้างตารางสรุปด้วยข้อมูลดังนี้ทุกครั้ง:

```
จำนวนนักศึกษาทั้งหมด:        ____ คน
จำนวนที่ส่งผลการประเมินครบ:  ____ คน
คะแนนเฉลี่ย (Mean):          ____
ส่วนเบี่ยงเบนมาตรฐาน (SD):   ____
คะแนนสูงสุด / ต่ำสุด:        ____ / ____

การกระจายเกรด:
  A:   __ คน (__%)   B+: __ คน (__%)
  B:   __ คน (__%)   C+: __ คน (__%)
  C:   __ คน (__%)   D+: __ คน (__%)
  D:   __ คน (__%)   F:  __ คน (__%)
  W/I: __ คน (__%)
อัตราผ่าน (≥D): ___%
```

### 1.2 การตรวจสอบความสอดคล้องกับแนวปฏิบัติสำนักวิชา

ตรวจสอบประเด็นต่อไปนี้ทุกข้อ และระบุ ✅ (ผ่าน) / ⚠️ (บกพร่องเล็กน้อย) / ❌ (ไม่ผ่าน) พร้อมคำอธิบายหลักฐาน:

**ก. สัดส่วนการประเมิน (90/10 Rule)**
- ตรวจสอบว่าสัดส่วนทักษะเฉพาะ : ทักษะทั่วไป = 90 : 10 ตามประกาศสำนักวิชา
- ตรวจสอบว่ามี Rubric จริยธรรม (5%) และทักษะสัมพันธ์ (5%) หรือทักษะทดแทนที่ระบุในกรณีไม่มีงานกลุ่ม
- สำหรับวิชาที่มีทั้งทฤษฎีและปฏิบัติ: ตรวจสอบสัดส่วนตามหน่วยกิตจริง (เช่น 3(2-2-5) → ทฤษฎี 60% : ปฏิบัติ 30%)

**ข. เกณฑ์การตัดเกรด**
- ระบุประเภทของรายวิชา: ทฤษฎี / ทฤษฎี+ปฏิบัติ / ปฏิบัติ/ภาคสนาม / S-U
- ตรวจสอบว่าใช้เกณฑ์ถูกประเภท:
  - ทฤษฎี: A≥80, B+75-79, B70-74, C+65-69, C60-64, D+55-59, D50-54, F<50
  - ปฏิบัติ/ภาคสนาม: A≥85, B+80-84, B75-79, C+70-74, C65-69, D+60-64, D55-59, F<55
  - S/U: S≥70, U<70
- หากเกณฑ์ที่ใช้ไม่ตรงกับประเภทวิชา ให้ระบุเป็น ❌ และคำนวณผลกระทบต่อการกระจายเกรด

**ค. ประเด็นปัญหาเฉพาะที่ต้องตรวจสอบ**
- Peer Assessment: มีนักศึกษาที่ได้ 0 คะแนนจากการไม่ประเมินหรือไม่ — ถ้ามี ระบุจำนวนและวิธีแก้ไขที่ใช้จริง
- Data Cleaning: มีคะแนนที่ผิดปกติ (outlier, null, รวมเกิน 100%) หรือไม่
- Mock-up Exam: หากมี ระบุว่าข้อสอบ mock-up แตกต่างจากข้อสอบจริงเพียงพอหรือไม่

**ง. กระบวนการทบทวนเกรด**
- ระบุว่ามีหลักฐานการผ่านกระบวนการทบทวน 3 ขั้นตอน (สาขา → คณะกรรมการห้อง → คณะกรรมการสำนักวิชา) หรือไม่

### 1.3 จุดเด่น / จุดอ่อน / ข้อเสนอแนะด้านการตัดเกรด

ให้วิเคราะห์และนำเสนอในรูปแบบตารางดังนี้:

```
┌─────────────────────────────────────────────────────────────────────┐
│ จุดเด่น (Strengths)                                                  │
│ • [ระบุสิ่งที่ดำเนินการได้ถูกต้องและเป็นแบบอย่างที่ดี]             │
│   หลักฐาน: [อ้างอิงข้อมูลจากเอกสาร]                                │
├─────────────────────────────────────────────────────────────────────┤
│ จุดอ่อน / ปัญหาที่พบ (Weaknesses / Issues Found)                    │
│ • [ระบุปัญหาแต่ละข้อ พร้อมระดับความรุนแรง: Critical/Major/Minor]   │
│   หลักฐาน: [อ้างอิงข้อมูลจากเอกสาร]                                │
├─────────────────────────────────────────────────────────────────────┤
│ ข้อเสนอแนะ (Recommendations)                                        │
│ • [ระบุการปรับปรุงที่ชัดเจน ปฏิบัติได้จริง มีกรอบเวลา]            │
│   ลำดับความเร่งด่วน: [ก่อนเปิดภาค / ระหว่างภาค / ภาคถัดไป]       │
└─────────────────────────────────────────────────────────────────────┘
```

**แนวทางการระบุจุดเด่นด้านการตัดเกรด (ตัวอย่าง):**
- การกระจายเกรดสมเหตุสมผล (ไม่กระจุกตัวที่ A หรือ F ผิดปกติ)
- ใช้เกณฑ์ถูกประเภท และสอดคล้องกับ CLO ที่ตั้งไว้
- มี Rubric มาตรฐานครบทั้ง 2 ด้าน (จริยธรรม + ทักษะสัมพันธ์)
- มีหลักฐาน Data Cleaning และกระบวนการทบทวนเกรดครบ 3 ขั้นตอน

**แนวทางการระบุจุดอ่อนด้านการตัดเกรด (ตัวอย่าง):**
- สัดส่วนไม่เป็น 90/10 หรือไม่ตรงกับโครงสร้างหน่วยกิต
- ใช้เกณฑ์ทฤษฎีกับวิชาที่เป็นปฏิบัติล้วน
- ไม่มี Rubric สำหรับ Peer Assessment ทำให้คะแนน 0 เกิดขึ้นโดยไม่มีแนวแก้ไข
- การกระจายเกรดผิดปกติ (เช่น F > 30% หรือ A > 80%) โดยไม่มีคำอธิบาย

## ส่วนที่ 2 — การประเมินคุณภาพรายวิชา (Course Specification Assessment)

**เอกสารที่ใช้:** มคอ.3 + มคอ.5 + ใบคะแนน + เอกสารอื่น ๆ ที่มีในโฟลเดอร์

### 2.1 การประเมินทีละหมวด (Section-by-Section Assessment)

ประเมินทุกหมวดด้วยรูปแบบ: **[สิ่งที่พบ] → [เกณฑ์ที่ควรเป็น] → [ระดับผ่าน/ไม่ผ่าน] → [คำอธิบาย]**

> **บังคับ:** ทุกหมวด (หมวดที่ 1 ถึง 6) ต้องปิดท้ายด้วยบทสรุปของหมวดนั้นเสมอ ในรูปแบบ:
> - **จุดเด่นของหมวดนี้:** [สรุปจุดเด่นที่พบในหมวดนี้ หรือ "ไม่พบ"]
> - **จุดที่ควรพัฒนาของหมวดนี้:** [สรุปจุดที่ควรปรับปรุง พร้อมระดับความรุนแรงหากมี หรือ "ไม่พบ"]
>
> บทสรุปรายหมวดนี้เป็นคนละส่วนกับบทสรุปภาพรวมในข้อ 2.3 ซึ่งยังต้องมีตามเดิม

#### หมวดที่ 1 — ข้อมูลทั่วไป

ตรวจสอบ:
- รหัส ชื่อวิชา (ไทย/อังกฤษ) ตรงกันและสมบูรณ์
- โครงสร้างหน่วยกิต X(A-B-C) ถูกต้องและสอดคล้องกับ มคอ.2
- ระบุอาจารย์ผู้รับผิดชอบและผู้ประสานงานครบ
- **ระบุสถานที่เรียนทุกแห่ง** (จุดที่มักพลาด)
- วันที่ปรับปรุงเป็นปัจจุบัน

#### หมวดที่ 2 — ลักษณะและการดำเนินการ

ตรวจสอบ:
- คำอธิบายรายวิชา (ไทย/อังกฤษ) สอดคล้องกัน ใช้คำกริยาระดับ ป.โท ไม่มีคำว่า "เบื้องต้น" หรือ "พื้นฐาน"
- วัตถุประสงค์ใช้คำกริยา Bloom's ที่วัดได้ ระดับ L4+ ทุกข้อ
- ระบุเหตุผลการพัฒนา/ปรับปรุงรายวิชาชัดเจน
- จำนวนหัวข้อสมดุลกับหน่วยกิต (1(0-2-1) → ≤4 หัวข้อหลัก; 2(1-2-3) → ≤6 หัวข้อ)

#### หมวดที่ 3 — CLO, PLO Alignment และการประเมิน

นี่คือหมวดที่ต้องวิเคราะห์ลึกที่สุด ให้ตรวจสอบทุกประเด็นต่อไปนี้:

**ก. คุณภาพ CLO แต่ละข้อ**

สำหรับ CLO แต่ละข้อให้วิเคราะห์:
```
CLO [X]: "[ข้อความ CLO]"
  - คำกริยา Bloom's: [ระบุ] → ระดับ: L[X] → เหมาะสมสำหรับ ป.โท: ใช่/ไม่ใช่
  - มีองค์ประกอบครบ 3 ส่วน (คำกริยา + เนื้อหา + บริบท): ใช่/ไม่ใช่
  - PLO ที่รองรับ: PLO[X] (Domain: [ความรู้/ทักษะ/จริยธรรม/ลักษณะ])
  - Domain ถูกต้อง: ใช่/ไม่ใช่
  - ประเด็น: [ระบุหากมีปัญหา]
```

**ข. PLO–CLO Mapping**

สร้างตารางแสดง Mapping และระบุ Gap:
```
| PLO | Domain | ระดับ PLO | CLO ที่รองรับ | น้ำหนัก% | Gap ที่พบ |
|-----|--------|-----------|--------------|----------|-----------|
| PLO1 | ความรู้ | L[X] | CLO1, CLO2 | XX% | [ระบุ/ไม่มี] |
```
- PLO ใดที่ไม่มี CLO รองรับ → ❌ Critical Gap
- CLO ระดับต่ำกว่า PLO มาก → ⚠️ Bloom's Gap

**ค. ความสอดคล้องระหว่างวิธีประเมินกับระดับ Bloom's**

```
| CLO | ระดับ Bloom's | วิธีประเมินที่ใช้ | เหมาะสม? | หมายเหตุ |
|-----|--------------|-----------------|---------|---------|
```
- L4 Analyze ที่ใช้เฉพาะ MCQ → ❌ ไม่เหมาะสม
- L5/L6 ที่ใช้เฉพาะ MCQ/SAQ → ❌ ไม่เหมาะสม
- วิชาปฏิบัติที่มี MCQ > 20% → ❌ ไม่ผ่านเกณฑ์

**ง. ประเภทการประเมิน (AAL/AFL/AOL)**

ระบุว่ามีการประเมินครบ 3 ประเภทหรือไม่:
- AAL (Self/Peer Assessment): [มี/ไม่มี] — ระบุรายการ
- AFL (Formative): [มี/ไม่มี] — ระบุรายการ (สอบกลางภาค, แบบทดสอบย่อย, รายงาน)
- AOL (Summative): [มี/ไม่มี] — ระบุรายการ (สอบปลายภาค, โครงงาน)

**จ. Constructive Alignment ภาพรวม**

ประเมินว่าแต่ละ CLO มีการเรียงร้อย PLO → CLO → วิธีสอน → วิธีประเมิน อย่างสอดคล้องหรือไม่

## ส่วนที่ 4 — แบบรายงานผลการทวนสอบผลลัพธ์การเรียนรู้รายวิชา (บังคับทุกรายวิชา)

> **ที่มา:** แบบฟอร์มนี้ใช้จาก `Evaluation template-ทวนสอบผลลัพธ์การเรียนรู้รายวิชา.pdf` ซึ่งเป็นแบบฟอร์มทางการของสำนักวิชาวิทยาศาสตร์สุขภาพ มฟล. ต้องแนบเป็นส่วนสุดท้ายของเอกสารประเมินทุกรายวิชา

### 4.1 โครงสร้างแบบฟอร์ม

แบบฟอร์มประกอบด้วย 4 ส่วน:
1. **ข้อมูลรายวิชา** — รหัส ชื่อวิชา อาจารย์ผู้รับผิดชอบ ภาค/ปีการศึกษา วันที่ทวนสอบ
2. **ตาราง 7 หัวข้อ** — คะแนน 3-2-1 พร้อมข้อดีและข้อพัฒนา
3. **สรุปผล** — ระดับ ควรปรับปรุง / ดี / ดีเยี่ยม
4. **ลายมือชื่อ 5 คน** — ผู้ทวนสอบภายใน, ผู้ทวนสอบภายนอก, อาจารย์ผู้รับผิดชอบ, ประธานสาขา, คณบดีหรือผู้แทน

### 4.2 เกณฑ์การให้คะแนน 7 หัวข้อ

**กฎสำคัญ: คะแนนทุกหัวข้อต้องอ้างอิงหลักฐานจากการวิเคราะห์ในส่วนที่ 1 และ 2 เท่านั้น**

| หัวข้อ | คะแนน 3 (ดีเยี่ยม) | คะแนน 2 (ดี) | คะแนน 1 (ควรปรับปรุง) |
|--------|------------------|-------------|---------------------|
| **1. ผลลัพธ์การเรียนรู้รายวิชา (CLO)** | CLO ครบทุก PLO ที่ได้รับมอบหมาย, คำกริยา Bloom's ชัดเจน L4+, ครบ 3 องค์ประกอบ | CLO ครบส่วนใหญ่ แต่บางข้อขาดองค์ประกอบหรือคำกริยาไม่ชัด | PLO บางตัวไม่มี CLO รองรับ หรือคำกริยาวัดไม่ได้ |
| **2.1 เนื้อหาของรายวิชา** | เนื้อหาครอบคลุม CLO ทุกข้อ มีคอลัมน์ CLO ในตารางสอน สมดุลกับหน่วยกิต | เนื้อหาครอบคลุมส่วนใหญ่ แต่ขาดการเชื่อมโยง CLO ในตาราง | เนื้อหาไม่สมดุลกับหน่วยกิต หรือไม่ครอบคลุม CLO หลัก |
| **2.2 วิธีการสอน** | วิธีสอนหลากหลาย สอดคล้องระดับ Bloom's ทุก CLO มีระบุรูปแบบต่อสัปดาห์ | วิธีสอนหลากหลาย แต่ไม่เชื่อมกับระดับ CLO ชัดเจน | วิธีสอนเดียว ไม่สอดคล้อง Bloom's หรือไม่ระบุรูปแบบ |
| **3.1 วิธีการวัดและประเมิน** | ครอบคลุมทุก CLO และ PLO, วิธีประเมินสอดคล้อง Bloom's | ครอบคลุมส่วนใหญ่ แต่บาง CLO/PLO ขาดการประเมิน | PLO หรือ CLO หลักไม่มีการประเมิน หรือ ceiling effect รุนแรง |
| **3.2 รูปแบบการวัด (AAL/AFL/AOL)** | มีครบทั้ง 3 รูปแบบ พร้อม Rubric เป็นลายลักษณ์อักษร | มีครบทั้ง 3 รูปแบบ แต่ขาด Rubric หรือเอกสารยืนยัน | ขาดรูปแบบใดรูปแบบหนึ่งหรือมากกว่า |
| **3.3 สัดส่วนคะแนน** | สอดคล้องประกาศสำนักวิชา (90/10) ครอบคลุมทุก Domain ของ PLO | สัดส่วน 90/10 ถูกต้อง แต่บาง Domain มีน้ำหนัก 0% จริง | ไม่เป็น 90/10 หรือ Domain สำคัญถูกละเว้นอย่างสมบูรณ์ |
| **3.4 คุณภาพข้อสอบ** | มี Item Analysis, ไม่มี ceiling/floor effect, มีหลักฐานปรับปรุง | มี Item Analysis บางส่วน หรือมี ceiling effect เล็กน้อย | ไม่มี Item Analysis, ceiling/floor effect รุนแรง, ไม่มีการปรับปรุง |

### 4.3 การคำนวณคะแนนรวมและระดับผล

```
คะแนนรวม = ผลรวมของทั้ง 7 หัวข้อ (คะแนนสูงสุด = 21)

ระดับผลการทวนสอบ:
  ≤ 14 คะแนน  →  ควรปรับปรุง  (ต้องดำเนินการแก้ไขก่อนเปิดสอนภาคถัดไป)
  15–18 คะแนน →  ดี            (ดำเนินการปรับปรุงตามข้อพัฒนาที่ระบุ)
  19–21 คะแนน →  ดีเยี่ยม      (รักษาระดับและเป็นแบบอย่างให้รายวิชาอื่น)
```

### 4.4 แนวทางการเขียนข้อดีและข้อพัฒนา

**ข้อดี (จุดเด่น):** อ้างอิงโดยตรงจากหลักฐานในส่วนที่ 1 และ 2 เช่น
- "CLO1 มีคำกริยา Bloom's L5 ครบ 3 องค์ประกอบ"
- "สัดส่วน 90/10 ตรงตามประกาศสำนักวิชาครบถ้วน"

**ข้อพัฒนา (จุดที่ต้องปรับปรุง):** ระบุปัญหาเฉพาะเจาะจง อ้างอิงจากหลักฐาน เช่น
- "PLO1 และ PLO4 ไม่มี CLO รองรับ" (ไม่ใช่ "CLO ควรปรับปรุง" เฉย ๆ)
- "Final Exam ceiling effect 69.4% ทำให้จำแนก CLO2 ไม่ได้"

### 4.5 ข้อกำหนดการแสดงผล

แบบฟอร์มในเอกสาร `.docx` ต้องมี:
- ตารางแสดงคอลัมน์ครบ: หัวข้อ | รายละเอียด | 3 | 2 | 1 | ข้อดี | ข้อพัฒนา
- เครื่องหมาย ● ในช่องคะแนนที่เลือก (ไม่ใช่ X หรือ ✓)
- แถวสุดท้ายแสดงคะแนนรวมต่อระดับ (เช่น "3×1 + 2×3 + 1×3 = 12") และคะแนนรวมทั้งหมด
- กล่องสรุปผลพร้อม ☑ ในช่องระดับที่ได้
- ตารางลายมือชื่อ 5 คน พร้อมเส้นบรรทัดและวันที่ (เว้นว่างไว้ให้กรอก)
- ข้อความท้ายสุด: "เอกสารนี้เป็นส่วนหนึ่งของกระบวนการทวนสอบผลลัพธ์การเรียนรู้รายวิชา ตามข้อกำหนดของ สป.อว. / AUN-QA Criterion 3"

## กฎทั่วไปในการวิเคราะห์ (General Rules)

1. **ใช้หลักฐานทุกครั้ง:** ทุกข้อสรุปต้องอ้างอิงจากเอกสารที่ได้รับ ระบุชื่อเอกสารและส่วนที่อ้างอิง
2. **ไม่เกินจากข้อมูล:** หากเอกสารใดไม่มีข้อมูลเพียงพอ ให้ระบุว่า "ไม่พบหลักฐานในเอกสารที่ได้รับ — ไม่สามารถประเมินได้"
3. **สมดุลระหว่างจุดเด่นและจุดอ่อน:** เอกสารที่ดีต้องระบุจุดเด่นที่มีหลักฐานรองรับอย่างน้อยเท่ากับจำนวนจุดอ่อน ไม่วิจารณ์ฝ่ายเดียว
4. **ข้อเสนอแนะต้องปฏิบัติได้จริง:** ทุกข้อเสนอแนะต้องระบุ: อะไร + ใคร + เมื่อใด + ผลที่คาดหวัง
5. **ใช้ภาษาวิชาการที่เหมาะสม:** เขียนด้วยภาษาไทยเป็นหลัก ใช้ภาษาอังกฤษสำหรับศัพท์เทคนิคที่ไม่มีคำแปลที่นิยม
6. **ร่าง มคอ.3 ต้องพร้อมใช้จริง:** ข้อความในร่าง CLO และการประเมินต้องเขียนสมบูรณ์ ไม่ใช่เพียงคำแนะนำในเชิงนามธรรม
7. **ความครอบคลุม:** อย่าข้ามหมวดหรือประเด็นใดแม้จะดูเล็กน้อย เพราะเอกสารนี้ใช้ในกระบวนการประกันคุณภาพอย่างเป็นทางการ
